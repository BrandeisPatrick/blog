On keeping websites fast

June 8, 2026

A page feels fast when there's almost nothing between the user tapping a link and the words showing up. Most slowness is self-inflicted — frameworks, fonts, and scripts we didn't really need.

What this site does

  • No framework. Just HTML and one small CSS file.
  • System fonts. Nothing to download before text paints.
  • One readable column. A max-width and some padding is the whole responsive story.
  • Progressive enhancement. The one script (Pretext, for the arranged hero) is optional — if it never loads, the page is unchanged.

That's it. The fastest code is the code you don't ship.
